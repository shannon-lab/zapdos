MessageFromInput

Print out a message from the input file

Description

This user object provides an option to print a message to the screen during the simulation. The message can be written in the input file. The user can customize when the message should print, the default option is to print it at the initialization of the simulation, with execute_on = INITIAL.

Example Input Syntax

[UserObjects]
  [message_out]
    type = MessageFromInput
    execute_on = timestep_end
  []
[]
(moose/test/tests/userobjects/message_from_input/message_from_input.i)

Input Parameters

  • messageThe message to print out

    C++ Type:std::string

    Unit:(no unit assumed)

    Controllable:No

    Description:The message to print out

    Default:INITIAL

    C++ Type:ExecFlagEnum

    Unit:(no unit assumed)

    Options:NONE, INITIAL, LINEAR, NONLINEAR_CONVERGENCE, NONLINEAR, POSTCHECK, TIMESTEP_END, TIMESTEP_BEGIN, MULTIAPP_FIXED_POINT_END, MULTIAPP_FIXED_POINT_BEGIN, FINAL, CUSTOM

    Controllable:No

    Description:The list of flag(s) indicating when this object should be executed. For a description of each flag, see https://mooseframework.inl.gov/source/interfaces/SetupInterface.html.

    Description:An optional suffix parameter that can be appended to any attempt to retrieve/get material properties. The suffix will be prepended with a '_' character.

    Description:For the old and older state use projected material properties interpolated at the quadrature points. To set up projection use the ProjectedStatefulMaterialStorageAction.
