IterationInfo

A Reporter object for tracking iteration information, such as the number of linear and nonlinear iterations. The "items" parameter controls the items computed. IterationInfo will declare reporters for each item requested.

Example Input Syntax

The following input file snippet demonstrates the use of the IterationInfo object.

[Reporters]
  [iteration_info]
    type = IterationInfo
  []
[]
(moose/test/tests/reporters/iteration_info/iteration_info.i)

Input Parameters

  • execute_onTIMESTEP_ENDThe list of flag(s) indicating when this object should be executed. For a description of each flag, see https://mooseframework.inl.gov/source/interfaces/SetupInterface.html.

    Default:TIMESTEP_END

    C++ Type:ExecFlagEnum

    Unit:(no unit assumed)

    Options:NONE, INITIAL, LINEAR, NONLINEAR_CONVERGENCE, NONLINEAR, POSTCHECK, TIMESTEP_END, TIMESTEP_BEGIN, MULTIAPP_FIXED_POINT_END, MULTIAPP_FIXED_POINT_BEGIN, FINAL, CUSTOM

    Controllable:No

    Description:The list of flag(s) indicating when this object should be executed. For a description of each flag, see https://mooseframework.inl.gov/source/interfaces/SetupInterface.html.

  • itemsThe iteration information to output, if nothing is provided everything will be output.

    C++ Type:MultiMooseEnum

    Unit:(no unit assumed)

    Options:time, timestep, num_linear_iterations, num_nonlinear_iterations, num_fixed_point_iterations

    Controllable:No

    Description:The iteration information to output, if nothing is provided everything will be output.

  • nl_sysnl0The nonlinear system that this should report iteration info for.

    Default:nl0

    C++ Type:NonlinearSystemName

    Unit:(no unit assumed)

    Controllable:No

    Description:The nonlinear system that this should report iteration info for.

  • prop_getter_suffixAn optional suffix parameter that can be appended to any attempt to retrieve/get material properties. The suffix will be prepended with a '_' character.

    C++ Type:MaterialPropertyName

    Unit:(no unit assumed)

    Controllable:No

    Description:An optional suffix parameter that can be appended to any attempt to retrieve/get material properties. The suffix will be prepended with a '_' character.

  • use_interpolated_stateFalseFor the old and older state use projected material properties interpolated at the quadrature points. To set up projection use the ProjectedStatefulMaterialStorageAction.

    Default:False

    C++ Type:bool

    Unit:(no unit assumed)

    Controllable:No

    Description:For the old and older state use projected material properties interpolated at the quadrature points. To set up projection use the ProjectedStatefulMaterialStorageAction.
