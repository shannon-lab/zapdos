FunctorKernel

This Kernel calculates the residual based on a functor. The form of the contribution is specified via the mode parameter.

If the ‘add’ option is selected for mode, this kernel simply adds the term from the functor value.

If, on the other hand, the ‘target’ option is selected for mode, this kernel sets the kernel variable (variable) weakly enforced to the value of a functor (functor).

commentnote

This kernel only imposes the equality between the variable and the functor (i.e. mode set to ‘target’) when it is the only kernel in that variable's equation on the specified subdomains.

The parameter "functor_on_rhs" determines the sign applied to the term. Since by convention in MOOSE, all terms are moved to the left hand side of the equation, a value of true for this parameter applies a sign of -1, and a value of false applies a sign of +1.

Input Parameters

  • functorFunctor to add. A functor is any of the following: a variable, a functor material property, a function, a postprocessor or a number.

    C++ Type:MooseFunctorName

    Unit:(no unit assumed)

    Controllable:No

    Description:Functor to add. A functor is any of the following: a variable, a functor material property, a function, a postprocessor or a number.

  • functor_on_rhsFalseIf true, the functor to add is on the right hand side of the equation. By convention, all terms are moved to the left hand side, so if true, a factor of -1 is applied.

    Default:False

    C++ Type:bool

    Controllable:No

    Description:If true, the functor to add is on the right hand side of the equation. By convention, all terms are moved to the left hand side, so if true, a factor of -1 is applied.

  • variableThe name of the variable that this residual object operates on

    C++ Type:NonlinearVariableName

    Unit:(no unit assumed)

    Controllable:No

    Description:The name of the variable that this residual object operates on

    C++ Type:std::vector<VariableName>

  • matrix_onlyFalseWhether this object is only doing assembly to matrices (no vectors)

    Default:False

    C++ Type:bool

  • modeaddThe operation mode, 'add' just returns the value of the functor and therefore adds the functor value to the residual'target' sets the residual term in such a way that the residual of the variable u vanishes if its value matches the given functor. Please note: In mode 'target' this kernel only imposes the equality between the variable and the functor when it is the only kernel in that variable's equation on the specified subdomains.

    Default:add

    C++ Type:MooseEnum

    Options:add, target

    Controllable:No

    Description:The operation mode, 'add' just returns the value of the functor and therefore adds the functor value to the residual'target' sets the residual term in such a way that the residual of the variable u vanishes if its value matches the given functor. Please note: In mode 'target' this kernel only imposes the equality between the variable and the functor when it is the only kernel in that variable's equation on the specified subdomains.
