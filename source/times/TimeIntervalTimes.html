TimeIntervalTimes

This object specifies times with a fixed time interval between a start time and end time. The start time and end time may be provided or if not, taken to be the start time and end time from the simulation (in this case, the executioner type must be Transient). If the end time does not fall exactly at the end of the end of the last time interval, then the end time is only included if the parameter "always_include_end_time" is set to true.

Input Parameters

  • time_intervalTime interval between times

    C++ Type:double

    Unit:(no unit assumed)

    Controllable:No

    Description:Time interval between times

Required Parameters

  • always_include_end_timeTrueIf true, includes the end time even if the last time interval would be partial

    Default:True

    C++ Type:bool

    Unit:(no unit assumed)

    Controllable:No

    Description:If true, includes the end time even if the last time interval would be partial

  • end_timeEnd time. If not provided, the simulation end time is used.

    C++ Type:double

    Unit:(no unit assumed)

    Controllable:No

    Description:End time. If not provided, the simulation end time is used.

    Default:TIMESTEP_END

    C++ Type:ExecFlagEnum

    Unit:(no unit assumed)

    Options:NONE, INITIAL, LINEAR, NONLINEAR_CONVERGENCE, NONLINEAR, POSTCHECK, TIMESTEP_END, TIMESTEP_BEGIN, MULTIAPP_FIXED_POINT_END, MULTIAPP_FIXED_POINT_BEGIN, FINAL, CUSTOM

    Controllable:No

    Description:The list of flag(s) indicating when this object should be executed. For a description of each flag, see https://mooseframework.inl.gov/source/interfaces/SetupInterface.html.

    Description:For the old and older state use projected material properties interpolated at the quadrature points. To set up projection use the ProjectedStatefulMaterialStorageAction.
