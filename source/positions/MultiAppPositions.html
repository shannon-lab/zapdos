MultiAppPositions

Obtain positions from MultiApps. This may only be used to set the positions of those same multiapps if an 'initial_positions' parameter is used.

Overview

MultiApps have a position that is used to translate their frame of reference compared to the main app frame of reference. This position can be used to align the meshes for the transfers or for outputting the MultiApps Exodus results at the translated positions.

warningwarning

This should not be used to generate new MultiApps. The MultiApps have already been created when the positions are gathered.

Example MultiApp File Syntax

In this example, the MultiAppPositions is obtaining the positions of three MultiApps.

[Positions]
  [multiapp]
    type = MultiAppPositions
    multiapps = 'm1 m2 m3'
  []
[]

[MultiApps]
  [m1]
    type = CentroidMultiApp
    input_files = 'multiapp_positions.i'
    cli_args = "MultiApps/active='';Positions/active=''"
  []
  [m2]
    type = FullSolveMultiApp
    input_files = 'multiapp_positions.i'
    cli_args = "MultiApps/active='';Positions/active=''"
  []
  [m3]
    type = FullSolveMultiApp
    input_files = 'multiapp_positions.i'
    positions = '0.2 0.4 1
                 0 0 6'
    cli_args = "MultiApps/active='';Positions/active=''"
  []
[]
(moose/test/tests/positions/multiapp_positions.i)

Input Parameters

  • multiappsName(s) of the multiapps providing the positions

    Unit:(no unit assumed)

    Controllable:No

    Description:Name(s) of the multiapps providing the positions

    Unit:(no unit assumed)

    Description:The list of flag(s) indicating when this object should be executed. For a description of each flag, see https://mooseframework.inl.gov/source/interfaces/SetupInterface.html.
